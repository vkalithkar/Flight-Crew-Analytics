## Introduction

We found a NASA dataset of physiological recordings from real pilot-copilot pairs experiencing various cognitive conditions in a flight simulator. The data captures 4 conditions: 1) a full realistic flight simulation (LOFT), 2) channelized attention (CA), a task designed to induce intense focus, 3) diverted attention (DA), a multitasking/distraction task, and 4) startle/surprise (SS), which is induced by a sudden event. For each pilot, four physiological signals were recorded simultaneously. Based on the data, we wanted to answer the following research question: **given these physiological signals alone, can a model learn to distinguish which cognitive states a pilot was in?**

This pipeline downloads physiological data from the NASA website, cleans and preprocesses two categories of physiological signals - first EEG and second ECG, GSR, and respiration - through separate tracks (MNE Toolkit and NeuroKit2, respectively). It engineers a sliding window of features from the cleaned signals, and trains an XGBoost classifier to predict which cognitive state a pilot was in during a flight simulation: channelized attention (CA), diverted attention (DA), startle/surprise (SS), or No Event.

## Quickstart

**1. Clone the repository**
```bash
git clone https://github.com/vkalithkar/Flight-Crew-Analytics.git
cd Flight-Crew-Analytics
```

**2. Set up the environment**
```bash
module load miniforge3
source "$(conda info --base)/etc/profile.d/conda.sh"
conda env create -f config/flight2-env.yaml
conda activate flight2
```

**3. Verify the environment**
```bash
python -c "import mne; print(f'MNE-Python version: {mne.__version__}')"
python -c "import xgboost; print(f'XGBoost version: {xgboost.__version__}')"
python -c "import sklearn; print(f'Scikit-learn version: {sklearn.__version__}')"
python -c "import neurokit2; print(f'NeuroKit2 version: {neurokit2.__version__}')"
```

**4. Run the pipeline**
```bash
sbatch pipeline.slurm
```

> **Note:** This pipeline runs on the W&M SciClone HPC cluster and requires SLURM. It will not run on a local machine as-is. Additionally, `02_eeg/00_get_bad_channels.py` only needs to be run once since it generates `pyprep_results.json` -- subsequent runs can skip it.

## Inputs and Outputs

**Inputs:**
- `data/data-site.htm` -- locally saved NASA webpage, which must be present before running the download script
- `data/raw/` -- raw CSVs downloaded from NASA, organized by pilot number, generated by the download script
- `data/eeg_preprocess/pyprep_results.json` -- bad channel dictionary generated by `00_get_bad_channels.py`, only needs to be generated once

**Final Outputs** (saved to `model/`):
- `model/best_xgb_model.pkl` -- the best XGBoost model found by grid search
- `model/model_stats.txt` -- a summary log containing the best hyperparameters, mean and per-fold train/test F1 scores across all 17 pilots, and the top 10 configurations ranked by test performance

## Script Details

The first step is to download the NASA data:

- **`/scripts/00_download_data.sh`** -- this bash script downloads, extracts, and cleans up the raw NASA data. It scrapes a locally saved HTML file of the NASA data page (`data/data-site.htm`), using `grep` to pull out every URL matching the `.zip` pattern and downloading each one into `data/raw/`. We also use rate-limiting flags (`--wait=15`, `--random-wait`) to avoid overwhelming the NASA server. Once all files are downloaded, the script unzips each one and deletes the original `.zip` files, leaving only the raw CSVs in `data/raw/`.

After downloading the data, the preprocessing is split into two tracks, one for ECG, GSR, and respiration (`/scripts/01_ecg_gsr_resp/`) and one for EEG (`/scripts/02_eeg/`).

- **`/scripts/01_ecg_gsr_resp/00_preprocess.py`** -- walks through the pilots' raw CSV files, cleaning signals for ECG, GSR, and respiration using NeuroKit2. We use `NEXUS_ECG_SCALE = 1.22e-4` to convert raw ECG values into proper millivolt units, and replace certain invalid event markers with 0. Three cleaning functions -- one per signal type -- all follow this pattern: convert signals to a numpy array, try the preferred NeuroKit2 cleaning method, and if it fails on a given file fall back to a basic Butterworth filter. Certain known bad pilot/signal combinations exist (pilots 5, 6, 10, 11, 18, and 21). Cleaned signals are saved to `data/ecg_gsr_resp_preprocess/`.

Because EEG measures electrical activity on the scalp, it picks up non-brain signals like eye blinks, jaw clenches, muscle movement, and heartbeats alongside actual neural activity. The following four scripts collectively remove this noise:

- **`/scripts/02_eeg/00_get_bad_channels.py`** -- identifies electrodes that are too corrupted to be included in the model. For each pilot and run, it: 1) reads the data CSV, 2) loads it into MNE (transposing from timepoints x channels to channels x timepoints), 3) renames channels to match MNE's naming convention so electrode locations are known, 4) applies frequency filters, 5) runs PyPrep to automatically detect bad electrodes, and 6) stores results in `data/eeg_preprocess/pyprep_results.json`.

- **`/scripts/02_eeg/01_interpolate_filter_reref.py`** -- reads the bad channel dictionary from `pyprep_results.json` and acts on it: reconstructing corrupted electrodes from neighboring ones, applying frequency filters, and re-referencing the signal to the average across all 20 electrodes (a standard EEG practice). Output is saved as `.fif` files -- MNE's native format, which preserves channel locations, sampling rate, and metadata.

- **`/scripts/02_eeg/02_ica.py`** -- removes artifacts that filtering alone cannot catch. Eye blinks, muscle movement, and heartbeats overlap in frequency with real brain activity, so ICA mathematically separates the signal into independent components. ICLabel (a pre-trained ML model) identifies which components represent artifacts and which represent brain activity. Those artifact components are removed and the cleaned signal is saved.

- **`/scripts/02_eeg/03_epoching.py`** -- cuts the continuous cleaned EEG signal into discrete 30-second windows for modeling. XGBoost has no concept of time, so rather than feeding it a raw time series, each window becomes one row of input with a single cognitive state label attached. Windows overlap by 24 seconds (advancing 6 seconds at a time) to maximize training samples from a small dataset of 17 pilots. The 30-second window size is driven by our slowest signal -- GSR requires 30 seconds to produce reliable features, so all signals conform to that constraint.

Now that all signals are cleaned, we produce visualizations:

- **`/scripts/03_eda_plots.py`** -- reads the cleaned ECG/GSR/respiration CSVs and produces time-series plots for each signal, with cognitive state periods shaded in color (SS=red, CA=yellow, DA=green). It detects state transitions by comparing each row's event code to the previous row's, then shades the resulting blocks. Generates 3 plots per file (respiration, ECG, GSR) saved to `eda_plots/{pilot_id}/`.

With all signals cleaned, the next step is to construct meaningful inputs for the model. Raw time series cannot be fed directly into XGBoost -- instead, each 30-second window is collapsed into a single row of features summarizing what happened during that period. Each feature represents a statistical or physiologically meaningful summary of one signal, with a single cognitive state label attached via majority vote.

- **`/scripts/04_feature_engineering.py`** -- extracts features from each 30-second window across all four signals using four dedicated functions (`eeg_features`, `ecg_features`, `gsr_features`, `rsp_features`). Each computes both generic statistics (mean, standard deviation, skewness, kurtosis) and signal-specific physiological measures -- heart rate variability metrics (RMSSD, SDNN) from ECG, tonic skin conductance level and response peak counts from GSR, and per-channel power from EEG. Subjects 10 and 21 are excluded due to pervasive bad signal data. Output is one feature CSV per pilot per run, saved to `data/feature_extracted/`.

- **`/scripts/05_model_creation_HPO.py`** -- loads all feature CSVs from `data/feature_extracted/` into a single dataframe, then trains an XGBoost multiclass classifier using Leave-One-Group-Out cross-validation (each fold holds out one pilot entirely, testing generalization to unseen people). A grid search over 648 hyperparameter combinations finds the configuration that maximizes weighted F1 score across all 17 folds (~11,000 total model fits). The best model is saved to `model/best_xgb_model.pkl` and a detailed stats log is saved to `model/model_stats.txt`.
